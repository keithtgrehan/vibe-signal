# Web Beta Signup Readiness

Date: 2026-06-03
Branch: `codex/ui-ux-product-design-research-upgrade`

## Current Readiness

The web beta surface is suitable for synthetic demo walkthroughs and recruiter review. It is not ready to collect live beta requests until legal review, real-device QA, and any intended submission backend are approved.

## What Is Ready

- Landing hero clearly explains communication-pattern support.
- Synthetic demo CTA is the primary first action.
- Six synthetic demo cards cover unclear asks, pressure/urgency, repair, low-signal fallback, boundary-respecting request, and overloaded messages.
- Result preview and result screen show evidence, pattern, limits, safe next step, and metadata-only feedback.
- Private paste flow requires explicit permission confirmation.
- Beta form is clearly local/static and does not submit data in this build.
- Privacy/legal access is visible from the top navigation.

## What Remains Blocked

- Live beta request submission.
- Real-device QA.
- Legal review of public beta copy and legal drafts.
- Screenshot set captured from synthetic examples.
- Final decision on whether beta requests should be email-only, backend-backed, or manually collected elsewhere.

## Demo Guidance

Use this flow for review:

1. Open the landing page.
2. Run a synthetic demo from the primary CTA.
3. Point to quoted evidence before interpretation.
4. Point to limits and safe next step.
5. Show metadata-only feedback gated by consent.
6. Return to input and show the private-text consent card.
7. Show the static beta form and explain that it does not submit data in this build.

## Safety Notes

- Do not capture private chat text in screenshots.
- Do not imply the beta form stores or sends requests.
- Do not present outputs as truth claims.
- Keep all demos synthetic.
